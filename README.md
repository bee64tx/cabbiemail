# Cabbiemail

A honeypot web application built on Cloudflare Pages with Pages Functions.

## Project Structure

```
cabbiemail/
├── public/                  Static assets served by Cloudflare Pages
│   ├── index.html           Main page
│   ├── 404.html             Custom not-found page
│   ├── _headers             Cloudflare Pages headers file (security headers)
│   ├── robots.txt           Crawler directives
│   └── assets/
│       ├── logo-source.png      Source logo (≥ 512×512)
│       ├── favicon-source.png   Source favicon (≥ 512×512)
│       ├── ASSETS.md            Asset pipeline documentation
│       └── fonts/               Downloaded web fonts (generated by build)
├── functions/               Cloudflare Pages Functions (edge backend)
│   └── api/
│       ├── auth.js          Authentication endpoint → /api/auth
│       └── csrf.js          CSRF token endpoint → /api/csrf
├── scripts/                 Build and tooling scripts
│   ├── build-assets.js      Asset optimisation pipeline (Sharp)
│   └── download-fonts.js    Google Fonts downloader
├── package.json
└── package-lock.json
```

## Prerequisites

- **Node.js ≥ 18** — required for build scripts and the Sharp image library
- **npm** — ships with Node.js
- **Cloudflare account** — needed for deployment (free tier is sufficient)

Wrangler is included as a dev dependency and does not need to be installed globally.

## Getting Started (Local Development)

1. Install dependencies:

   ```sh
   npm install
   ```

2. Place source images in `public/assets/` — you need `logo-source.png` and `favicon-source.png` (minimum 512×512 px). These are already checked in.

3. Run the full build (downloads fonts from Google Fonts + optimises/resizes images):

   ```sh
   npm run build
   ```

4. Start a local dev server with Pages Functions support:

   ```sh
   npm run dev
   ```

5. Open `http://localhost:8788` in your browser.

### Scripts

| Command                  | Description                                              |
|--------------------------|----------------------------------------------------------|
| `npm run dev`            | Start local dev server (Wrangler Pages, port 8788)       |
| `npm run build`          | Full production build (fonts + assets)                   |
| `npm run build:assets`   | Optimise and resize source images only                   |
| `npm run download:fonts` | Download web fonts from Google Fonts                     |

## Deploying to Cloudflare Pages

### Option A: Dashboard (Git Integration)

1. Log in to the [Cloudflare Dashboard](https://dash.cloudflare.com/) → **Pages** → **Create a project**
2. Connect your Git repository
3. Configure build settings:
   - **Build command:** `npm run build`
   - **Build output directory:** `public`
   - **Root directory:** `cabbiemail` (the project is nested inside a monorepo)
4. Click **Deploy**

### Option B: Direct Upload via Wrangler CLI

```sh
# Create the Pages project (first time only)
wrangler pages project create cabbiemail

# Build
npm run build

# Deploy
wrangler pages deploy public/ --project-name cabbiemail
```

### Pages Functions

The `functions/` directory is automatically detected by Cloudflare Pages. Files map to URL paths based on their filesystem location:

- `functions/api/auth.js` → `/api/auth`
- `functions/api/csrf.js` → `/api/csrf`

No `wrangler.toml` or bindings configuration is needed — the project is entirely stateless.

## Cloudflare Dashboard Settings

- **Custom domain** — add via Pages project → Custom domains (optional)
- **Compatibility date** — set to a recent date (e.g. `2024-01-01`) in Pages project → Settings → Functions → Compatibility date
- **Environment variables / secrets** — none required; the app is stateless
- **KV, D1, R2, or other bindings** — none required

## Security Headers

The file `public/_headers` is automatically applied by Cloudflare Pages to all responses. It sets:

- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: no-referrer`
- `Permissions-Policy` (disables geolocation, microphone, camera, payment)
- `X-XSS-Protection: 1; mode=block`
- `Content-Security-Policy` on `/` (restricts sources to `'self'`, allows inline styles and scripts)

## Conventions

- All user-facing copy is written in **British English** (e.g. colour, favour, organisation).
- The project targets Cloudflare Pages; there is no traditional server.
- Functions in `functions/` are deployed as Pages Functions at the corresponding URL paths.
